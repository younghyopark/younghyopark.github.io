1. Introduction

In Ubuntu, there's a long-running joke: "less is more." That's because the less command, despite its name, actually offers more functionality than more.

Less is a program similar to more (1), but it has many more features. Less does not have to read the entire input file before starting, so with large input files it starts up faster than text editors like vi (1).

A similar paradox shows up in imitation learning.

In theory, imitation learning should benefit from more data. But in practice, larger datasets can hurt performance. Meanwhile, carefully curated, smaller datasets often yield surprisingly better results.

This contradiction has sparked growing interest in data curation for imitation learning. Rather than scaling data collection blindly, recent work asks: What data is actually helpful? How do we identify and retain valuable demonstrations while filtering out detrimental ones?
